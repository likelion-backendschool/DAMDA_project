## 현재까지 개발 과정 요약 (최소 500자 이상)

---

- 정혜수
    
    **진행 완료**
    
    - 리뷰 CRUD 기능 구현
    
    **진행 예정**
    
    - 오늘 의논 결과대로 태그 변경 후 조회 기능 구현 필요
        - SPOT 리스트에서 태그 조회할 수 있도록
        - 리뷰 상세페이지에서 태그 조회할 수 있도록
    - USER MERGE 후 리뷰 및 리스트에서 글쓴이 조회할 수 있도록 정리
    - 리뷰 리스트 2개로 분리 필요
        - SPOT별로 작성된 리스트
        - USER별로 작성된 리스트
    
- 장동익
    - 로그인 UI 수정 완료
    - 오류 수정 완료
    - 초대 버튼 구현 후 여행지 초대 로직 구현 중
        - 초대 링크를 담는 엔티티 생성
        - 초대 링크의 유효기간 추가할 것
        - 최초 생성자만 여행계획 삭제할 수 있도록(생각중)
        - 장바구니에 담는 여행지가 실시간으로 보이도록 ajax로 구현(생각중)

    - 여러개의 장바구니에 동일한 여행지 들어가지 않는 문제 수정

## 개발 과정에서 나왔던 질문 (최소 200자 이상)

---

- SPOT - REVIEW
    - 여행 시작 일, 여행 종료 일 변수의 필요성 → 봄 , 여름, 가을, 겨울 태그로 날씨의 의미 표현
    - SPOT 리스트에서 REVIEW 일부 조회 방식 → 제목만 조회 가능
    - SPOT 리스트에서 다량의 REVIEW 스크롤로 조회 → VIEW MORE REVIEWS 버튼으로 SPOT별 리뷰 리스트로 이동
    - 리뷰 리스트 정렬 시 _인식 불가 문제 → 변수 명 변경 합의
    - 리뷰의 댓글 기능 → 최소 목표 달성 후 댓글과 대 댓글까지 업데이트 예정
    - 여행지 추가 페이지와 관리자 페이지 필요성 논의 → 추후 천천히 업데이트 예정
